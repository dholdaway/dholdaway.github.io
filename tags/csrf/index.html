Critical Vulnerability: WordPress 404 Error Monitor CSRF Attack (CVE-2024-11118)

Published: Saturday, November 16th, 2024 at 04:15:06 UTC Generated: November 16th, 2024

Summary

A critical vulnerability (CVE-2024-11118) has been identified in the 404 Error Monitor plugin for WordPress, affecting all versions up to and including 1.1. The vulnerability is a Cross-Site Request Forgery (CSRF) issue due to missing or incorrect nonce validation on the updatePluginSettings() function.

Executive Summary

This vulnerability allows unauthenticated attackers to manipulate plugin settings and clear error logs by tricking site administrators into performing an action, such as clicking on a link. The critical flaw lies in the 404 Error Monitor plugin’s failure to properly validate nonces (a measure to prevent CSRF attacks) in the updatePluginSettings() function.
